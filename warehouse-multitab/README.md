# Warehouse Multitab

نسخة عربية RTL احترافية متعددة التبويبات لنظام إدارة المخزون، مبنية على الحزمة الآمنة ذاتها في `warehouse-one-tab`.

## التبويبات

- **لوحة المتابعة:** المؤشرات، أعلى الأرصدة، وتوزيع المخزون حسب الجهة المالكة.
- **الأصناف والأرصدة:** البحث والفلاتر وجدول الأرصدة وإضافة صنف جديد.
- **الحركات والأذونات:** تسجيل الوارد والصادر ومراجعة سجل التدقيق والتصحيح والعكس.
- **التقارير:** التقرير الإداري الرسمي وتقارير PDF وتصدير CSV.
- **الإدارة:** إدارة المستخدمين والأدوار والنسخ الاحتياطية، وتظهر للمدير فقط.

التبديل بين التبويبات يخفي الأقسام ويظهرها من دون إعادة إنشاء النماذج، لذلك تبقى البيانات غير المرحلة في نموذج الحركة ونموذج الصنف محفوظة أثناء التنقل.

## الصلاحيات والتشغيل

تستخدم النسخة نفس تسجيل الدخول والجلسات والأدوار `ADMIN` و`STOREKEEPER` و`AUDITOR` ونفس التحقق الخادمي الموجود في النسخة ذات التبويب الواحد. ملفات الخادم وملف البيان وملفات `clasp` منسوخة كما هي دون تعديل.

يعرض جدول الأصناف 20 صفاً افتراضياً، مع خيارات 20 و50 و100.

يدعم تبويب الأصناف استيراد ملفات `.xlsx` و`.xls` و`.csv` بعد عرض معاينة كاملة. يمكن الاختبار بملف `../sample-data/catalog-76.csv`، وتُحفظ النتائج في ورقة `Items`.

## الرفع إلى Google Apps Script

1. انسخ `.clasp.json.example` إلى `.clasp.json` داخل هذا المجلد.
2. ضع معرّف مشروع Apps Script الحقيقي بدلاً من القيمة التجريبية.
3. نفّذ `clasp status` وتأكد أن الظاهر هو ملفات التشغيل فقط.
4. نفّذ `clasp push` ثم أنشئ إصدار Web App جديداً.

لا ترفع ملف `.clasp.json` الحقيقي إلى GitHub لأنه يحتوي معرّف مشروعك.
